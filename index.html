Classroom Communication Aide

The Project

The Classroom Communication Aide is an academic project developed CSCI 436/536: Technology for Social Good at Western Washington University. The class was taught by Dr. Shameem Ahmed in Fall 2025. Our mission is to create an ICT to address SDG 4 by bridging the communication gap between multilingual K-12 learners and their educators.

The Problem

In todays educational system, language barriers present significant challenges to effective classroom communication. Teachers often struggle to convey information to students who speak different primary languages, while students may feel isolated or unable to express their needs and questions. Some districts have resources in place to provide support to ELL's, but many do not. As a result students spend a majority of their time unassisted in a general classroom.

As of fall 2021, there were 5.3 million ELL students, 10.6% of all students in the United States. Outside of the United States, there are a great number of students who deal with being native language learners. The frameworks for this project can be extended to provide solutions for education across the world.

We are developing this solution for K-12 teachers and their ELL students. We believe this solution would be a great tool that would improve the learning experience of ELL students, especially in regions or schools that lack existing resources.

Our Solution

We decided to develop an accessible web application that could be used across a wide range of devices. We developed our solution to focus on using visual aids, to begin introducing the association between objects and their pronunciations. We focused on improving the interactions between multilingual students and their teachers, as we decided to implement real time two-way translation to help facilitate communications.

Our application will support essential requests such as "I need water" or "I don't understand" with the use of visual aids to help student build language association. We will create a default set of these requests to ensure a clear signal can be sent to the teachers end of the application to resolve. Teachers will have the ability to prepare and share specific materials for ELLs, create shortcut sets for different classroom activities, and receive notifications when a student needs assistance.

Our solution will differ from existing ones as we will have extensive support for language translation when needed. Many existing solutions exist that allow a student teacher connection for sending requests, but generally lack integrated translation tools for one-on-one interaction between the student and teacher.

We focused on developing a web application as opposed to a native application as one of our main goals was compatibility. Further, a website would be the most simple solution to host and manage as we would not need to work on developing an application to reach an app store. We used a suite of technologies that our team was familiar with and we felt would best fit the needs of our application. We developed with HTML, CSS, JS specifically using React and Tailwind for a fast component based user interface. For message communication we used WebSockets for low latency and notifications. Our backend API is implemented with Node.js to handle our business logic and integrations, with Supabase to host our PostgreSQL database and manage user authentication.

Systematic Literature Review

We have to read two paper each that are relevant to our project. In total we have eight papers. For each paper, we need to answer the following questions:

  • What is the research goal of this paper?
  • What are the major contributions of this paper?
  • How does your project relate to this paper?
  • How does your project differ from this paper?

    User Study

    We interviewed a total of four multilingual learner specialists and educators, many working locally in the elementary schools of Bellingham.

    Following a questionnaire that we drafted as a team, we were able to have in-depth discussions that gave us helpful insights on the current state of technology use in classrooms, the challenges that ELL students face, and what kind of support already exists.

    A few findings:

  • Beyond language barriers, ELL students may experience cultural barriers that can impact their feeling of belongingness or their connection to their teacher. Representation is an important thing to keep in mind.
  • School districts provide Google Chromebooks to students.
  • There are often hand signals used in class for needs to prevent interrupting the class.
  • ELL students start at a wide variety of levels, from kids who have never been in school or know how to read in their native language, to those who learn fast. Their initial understanding, as well as how fast they learn English, is very different for every kid. Teachers must handle many different changing levels at once.
  • From the perspective of teachers, the use of translation tools such as Google Translate go a long way when working with ELLs in the classroom. Integrating a tool like this into an application would be a huge benefit.
  • Existing tools struggle to incorporate a full suite of features. In other words, applications exist to provide singular functions in the classroom, but there are not many solutions that integrate multiple functions.

    Final Product

    Login Page
    The login page for the website is accessed from the landing page. We have two login methods, one with an email and password and the other using Google OAuth. When a user registers an account, either with OAuth or email, they will need to complete their account registration with a username and eventually other fields which will be used within the application.
    Student Page:
    The main student page provides access to every feature that the student may need, all in one place. The activity header describes the current class activity and helps the student better understand what is going on around them. The command buttons allow the student to send basic needs requests directly to their teacher. They can add and edit the appearance of certain buttons using the edit button. The student can use also use the message bar to send a private and unique message to their teacher, and use the translator button to open up the translator page.
    Translator Page:
    Allows students to translate between their language and their teacher's language. It allows them to input using speech and listen to the translation. There is a help button to explain how to use the features on that page in their own language.
    Teacher Page:
    Teachers will have a queue of incoming messages from students. They can quickly allow a request or send a response back. They can also set an activity status and subject, which is set on all students' pages. The teacher has sub pages where they can view and sort a log of all requests made in a class, and a student page that displays information about each student, including their language, what messages, and usage frequency. Thereâ€™s also a profile page, where a teacher can register their class to get a unique class code for students to register with.
    Teacher Custom Response Message Page:
    Allows teacher to reply to a request through a custom message. It will display the student name, requested test, request time, and date. Once typed the message is typed in the text box, the teacher can click send to send the message. The teacher will go back to the previous page once they click send, cancel, or the go back button.
